RTX 5080 和 RTX 3090 组合在 Qwen 3.6 27B Q8 上实现 80+ tokens/秒

⭐ 8.0/10

一篇博客文章报道，使用 RTX 5080 和 RTX 3090 的双 GPU 设置，在 Q8 量化下运行 Qwen 3.6 27B 模型，实现了每秒超过 80 个 token 的速度。 这表明使用消费级多 GPU 配置可以实现高性能的本地 LLM 推理，可能减少在编程和写作等任务中对云 API 的依赖。 Qwen 3.6 27B 是一个稠密 27B 参数模型，在编程基准测试上表现优于更大模型，而 Q8 量化在速度和质量之间取得了平衡，质量损失极小。

iMil · 6月13日 09:55

背景

像 Qwen 3.6 这样的大语言模型需要大量 GPU 内存和算力。量化通过使用 8 位整数（Q8）代替 16 位浮点数来减小模型大小，大致将内存使用减半，同时保留大部分准确性。跨多个 GPU 运行模型可以分摊工作量，实现更快的推理。

💬 社区讨论

评论者分享了 Qwen 3.6 的调参建议（例如`--temp 1.0 --top-p 0.95 --top-k 20`），并指出针对 Nvidia 硬件推荐的推测解码设置应使用`--spec-draft-n-max 2`而非 3。一些用户将性能与其他硬件（如 Tenstorrent 卡）进行了比较，另一些用户则指出加州电费成本使得本地设置在与云 API 竞争中不具优势。

在家低成本 AI 编程：自托管开源模型

⭐ 8.0/10

Stephen Bochinski 发布了一篇实用指南，介绍如何在家使用 AI 编程而不产生高昂的 API 费用，提倡在本地硬件上自托管开源模型。 该指南满足了人们对成本效益高的 AI 编程解决方案日益增长的需求，尤其是在开发者寻求昂贵订阅服务（如 Codex 和 Cursor）替代方案的情况下。它强调了自托管在隐私保护和长期节省方面的可行性。 文章承认自托管需要大量的硬件前期投资，且模型能力弱于前沿实验室，但指出对于有持续长时间任务的用户来说，这是值得的。社区评论还提到通过 Opencode 工具使用 Deepseek 的 API 作为低成本替代方案。

sbochins · 6月13日 16:45

背景

AI 编程助手如 GitHub Copilot、Cursor 和 Codex 提供代码生成和补全功能，但通常按 token 或按月收费。自托管是指在本地 GPU 上运行开源 LLM（如 Qwen、CodeLlama），从而消除持续的 API 费用。最近的模型如 Qwen3-Coder-Next 可在单块 24GB GPU 上运行，并达到具有竞争力的基准测试成绩。

💬 社区讨论

评论者分享了不同体验：有人认为每月 100 美元的 Codex 计划足够，而有人质疑用户如何在 Cursor 上花费那么多。一位评论者指出自托管并非完全免费，因为电费成本，但重视隐私溢价。另一位推荐通过 Opencode 使用 Deepseek 的 V4 flash 模型，几个月仅需 10 美元。

美国人口普查局禁止在统计产品中使用噪声注入

⭐ 8.0/10

美国人口普查局已禁止在其所有统计产品中使用噪声注入（即差分隐私），推翻了 2020 年人口普查采用的关键隐私保护措施。 这一政策变化可能削弱公众对政府数据收集的信任，并增加在已发布统计数据中重新识别个人的风险，影响依赖准确且隐私保护的人口普查数据的研究人员、政策制定者和公众。 该禁令适用于人口普查局发布的所有统计产品，而不仅仅是十年一次的人口普查。批评者认为，没有噪声注入，敏感信息可能从汇总统计中被重建，尤其是当与其他数据源结合时。

nl · 6月13日 13:54

背景

统计披露控制（SDC）包括防止从已发布数据中识别个人的技术。噪声注入是差分隐私的一种形式，通过添加随机噪声来掩盖个体贡献。人口普查局曾在 2020 年人口普查中实施差分隐私以解决隐私问题，但这一决定推翻了该做法。

💬 社区讨论

评论者表达失望和担忧，强调对人口普查过程的信任本就不高，移除隐私保护可能导致数据滥用。一些人建议在分析阶段而非原始数据中添加噪声，另一些人则担心国家安全影响。
